# operation/sessions/ — сессии (атом = Session Note)

Каждая значимая рабочая сессия — **один файл**: атом с провенансом («что-где-когда») и одновременно краткая запись сессии. Видно, **что откуда взялось**: какие концепты и решения родились из какой работы. Канон «атом = Session Note» (2026-07-17) — спущен по лестнице DM-EXO-8; в Ex-Base слой **опционален** (для микро-действий не обязателен).

Слой ведут навыки `/start-session` (создаёт атом) и `/close-session` (финализирует и проставляет `produced:`); можно и руками — формат ниже. Чистый Markdown, без плагинов.

## Имя файла

`operation/sessions/S{ГГГГММДД}T{ЧЧММ}-{slug}.md` — напр. `S20260713T0930-first-concepts.md`. `{slug}` — латиница kebab-case, max 40 символов.

## Шаблон атома (упрощённый профиль Base)

```markdown
---
# — атом-провенанс —
type: session-note
session_id: S20260713T0930-first-concepts
when: 2026-07-13T09:30
where: claude-code          # среда: claude-code | cowork | cursor | obsidian
project:                    # slug из projects/ (или пусто)
mode: ispolnenie            # в Ex-Base режим один — Исполнение (фиксировано)
source: dialogue            # dialogue | document | transcript | voice
produced: []                # карточки concepts/, рождённые в сессии (провенанс-выход)
# — жизненный цикл —
Status: "🔵 Активна"        # 🔵 Активна | 🟢 Выполнена | ⚪ Отложена | 🔴 Не сработало
Active: true
---

-

```

## Лестница (совместимость Base ⊂ Pro ⊂ Master)

Имена полей frontmatter и заголовки секций **едины по лестнице DM-EXO-8**: это подмножество полного профиля Ex-Pro. Апгрейд до Pro = добавление полей (`StatusCode/EndTime/DurationMin`) и секций (Открытые векторы, Хронология, ★ Insights, 🔁), **без переименований** — существующие атомы остаются валидными.

## Правила

- **Сессия ≠ журнал дня.** Хронология дня — в `../daily/`; сессия ссылается на день, день — на сессию.
- Провенанс: карточка концепта, рождённая в сессии, получает `source_session:` с её `session_id`; сессия перечисляет карточки в `produced:`.
- Сессия — путь по умолчанию для значимой работы, но не обязательна для микро-действий.
