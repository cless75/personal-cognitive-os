# Навык: инициация репозитория (прямая команда)

Разворачивает свежий Personal Cognitive OS и доводит владельца до первого рабочего цикла. Предназначен для запуска в Claude Cowork/Code **до** того, как репозиторий существует, поэтому распространяется как глобальный навык (`~/.claude/skills`).

## Протокол (через AskUserQuestion)

### Шаг 1 — Параметры (AskUserQuestion)
Спроси:
- **Способ развёртывания:** «Клонировать публичный seed с GitHub» *(рекомендуется, получишь апдейты)* / «Развернуть из встроенного шаблона» *(оффлайн)*.
- **Каталог:** куда развернуть (предложи разумный дефолт, например `~/personal-cognitive-os`).
- **Имя репо** (если не дефолт).

### Шаг 2 — Развёртывание
- **Clone:** `git clone <seed-url> <каталог>` → удали историю seed при желании (`rm -rf .git && git init`), либо оставь как fork-базу.
- **Scaffold (оффлайн):** создай структуру из шаблона (README, START-HERE, CLAUDE.md, concepts/, About-Me/, sources/, projects/, daily/, onboarding/, .claude/, manifest.yaml, .gitignore).
- Подставь токены `{{learner}}`, `{{learner_email}}`, `{{date}}` в `manifest.yaml`, `About-Me/about-me.md`, `CLAUDE.md`.
- `git init` (если новый) + первый commit.

### Шаг 3 — Персонализация
Запусти логику навыка `init-me` (заполни `About-Me/about-me.md` — из файла или коротким диалогом).

### Шаг 4 — Карта проектов
Убедись, что в `CLAUDE.md` есть пустой блок `<!-- projects-map:start -->…<!-- projects-map:end -->`.

### Шаг 5 — Первый шаг
Предложи: «Положи свой первый документ/транскрипт в `sources/` и запусти `/review-concepts`». Дай ссылку на `onboarding/01-first-review.md`.

## Заметка о среде
- **Claude Code:** после развёртывания репо-локальные навыки в `.claude/skills` подхватываются автоматически.
- **Cowork:** установи навыки репо как плагин (`.claude-plugin/`) или ZIP — см. `onboarding/surfaces.md`.
