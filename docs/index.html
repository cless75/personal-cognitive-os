Установка Exocortex Base

Как поставить и запустить свой экзокортекс за несколько минут.

Репозиторий: github.com/cless75/personal-cognitive-os

Что нужно

  • Claude Code ИЛИ Claude Desktop с Cowork — рабочая среда, где живут навыки.
  • git — чтобы скачать репозиторий.
  • Опционально Obsidian — если хочешь просматривать концепты как связанный граф (только просмотр).

Какой ты пользователь?

Найди свою ситуацию — и переходи к нужному сценарию.

СитуацияСценарий
Хочу личный экзокортекс с нуляA
Уже есть проект без seed, хочу добавить экзокортексB
Просто посмотретьObs

A Начать с нуля

Ты заводишь свежий экзокортекс. Всё, что нужно, — скачать репо и подключить навыки.

A1Claude Code

git clone https://github.com/cless75/personal-cognitive-os
cd personal-cognitive-os

Открой папку в Claude Code — команды доступны сразу (Claude Code сам читает .claude/skills). Переходи к разделу «Первый рабочий цикл».

A2Cowork

git clone https://github.com/cless75/personal-cognitive-os
cd personal-cognitive-os

Открой папку в Cowork → Settings → Plugins → Install plugin from folder → выбери .claude-plugin/.

Важно

Cowork не читает .claude/skills из папки сам (в отличие от Claude Code) — поэтому навыки подключаются плагином. Это делается один раз. Дальше переходи к разделу «Первый рабочий цикл».

B Добавить в существующий проект (без seed), недеструктивно

У тебя уже есть проект, и ты хочешь встроить в него экзокортекс, ничего не сломав. Команда /init-exocortex (режим B) сама подтянет seed, встроит навыки и шаблоны, создаст недостающие папки и допишет карту проектов в CLAUDE.md — не трогая твои существующие файлы.

B1Claude Code

Сначала один раз поставь init-exocortex глобально (эта команда нужна до того, как в проекте появится seed):

Windows PowerShell:

git clone --depth 1 https://github.com/cless75/personal-cognitive-os "$env:TEMP\pco"
New-Item -ItemType Directory -Force "$HOME\.claude\skills" | Out-Null
Copy-Item -Recurse -Force "$env:TEMP\pco\.claude\skills\init-exocortex" "$HOME\.claude\skills"
Remove-Item -Recurse -Force "$env:TEMP\pco"

macOS / Linux / Git Bash:

git clone --depth 1 https://github.com/cless75/personal-cognitive-os /tmp/pco
mkdir -p ~/.claude/skills && cp -r /tmp/pco/.claude/skills/init-exocortex ~/.claude/skills/ && rm -rf /tmp/pco

На Windows ~/.claude/skills = C:\Users\<ты>\.claude\skills.

Затем в своём проекте запусти:

/init-exocortex        # выбери режим B

Команда недеструктивна: добавляет только недостающее и дописывает карту проектов в CLAUDE.md, не перезаписывая существующие файлы.

B2Cowork

Поставь навыки (плагином из .claude-plugin/ или ZIP через Settings → Capabilities → Skills → Upload), затем запусти:

/init-exocortex        # выбери режим B

Просмотр в Obsidian

Хочешь просто посмотреть содержимое, без Claude?

  • В Obsidian: Open folder as vault → укажи папку репозитория.
  • Это чистый Markdown — плагины не нужны.
  • Папка concepts/ красиво раскрывается как граф связей между концептами.

Первый рабочий цикл (общий)

Этот цикл одинаков для всех сценариев. Навык /review-concepts — редактор, а не сочинитель: он сохраняет твой авторский голос.

/init-me                # заполни профиль владельца About-Me/about-me.md
# положи документ или транскрипт в sources/
/review-concepts        # авторское ревью → карточка концепта в concepts/

Что делает /review-concepts:

  • извлекает концепты из твоего материала;
  • показывает draft с твоей цитатой (первоисточник формулировки);
  • задаёт 3–7 вопросов по слоям: ядро / границы / связи / имя;
  • прогоняет 4 guard'а качества: category-error, forced-symmetry, substrate, false-second-axis;
  • и только по твоему явному вердикту оформляет карточку в concepts/.

Дальше — проекты:

/new-project → /start-session → /close-session

Карта проектов при этом сама пополняется в CLAUDE.md (блок между маркерами projects-map), который Claude видит в каждой новой сессии.

Обновиться

/update-exocortex        # или git pull, если у тебя форк

Твои концепты, профиль и проекты остаются нетронутыми — обновляется только каркас.

Если что-то не так

  • Cowork не видит команды — это норма, а не баг. Cowork не читает .claude/skills из папки автоматически. Поставь навыки плагином (.claude-plugin/) или через ZIP Upload.
  • CRLF-warnings при clone/commit на Windows — безобидны. git просто предупреждает о переводе окончаний строк LF↔CRLF; на содержимое это не влияет.
  • About-Me/ приватный — это твой личный профиль. В форк его можно не коммитить (добавь About-Me/ в .gitignore), чтобы личный контекст не уходил в публичный репозиторий.
