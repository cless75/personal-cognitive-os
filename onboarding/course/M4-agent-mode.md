# M4 · Агентный режим: Cognitive OS с агентом

> **Столп B — работа на платформе.** Self-paced: прочитай гайд → разверни репо на своей машине → пройди self-quiz → зафиксируй deliverable. ~50 мин. Нужен установленный git и агент (Claude Code / Cursor / Cowork). **Это ядро курса** — здесь ты реально разворачиваешь свой экзокортекс. [Карта курса](00-start.md).

---

## Слой 1 · Сущность (узнать)

**Агентный режим** — способ жить в своём Cognitive OS, где знание лежит не в базе данных платформы, а в **отдельном git-репозитории `personal-cognitive-os`** на твоей машине. С этим репо работает агент — Claude Code, Cursor или Cowork.

Ключевое отличие от веб-режима ([M3](M3-web-platform.md)): здесь нет плагинов и бэкенда. Всё — **чистый git + Markdown**. Репо клонируется, коммитится, переносится обычными git-командами. Именно поэтому агентный режим даёт то, чего не даёт веб: **отчуждаемость**. Сделал `git clone` — и всё твоё знание уехало с тобой на любую машину, к любому агенту.

M4 отвечает на три вопроса:
1. **Из чего** состоит репо (зоны + системная `.exocortex/`).
2. **Чем** ты им управляешь (набор команд агента).
3. **Как** он растёт и переносится (лестница зрелости + мост отчуждаемости).

---

## Слой 2 · Концепты-принципы (понять)

### 2.1 Репо `personal-cognitive-os`: чистый git + Markdown

Инвариант агентного режима: **всё — чистый git и Markdown, без плагинов и бэкенда**. Никакой БД, никакого сервера. Синхронизация знания идёт через git (`push`/`pull`), а не через файловый облачный клиент.

Отсюда главное свойство — **vendor-independence:** LLM находится **снаружи** экзокортекса, за ИИ-Мембраной. Движок резонинга сменный: сегодня один агент, завтра другой — само знание в репо не меняется. Репо не привязано ни к вендору модели, ни к конкретному агенту.

### 2.2 Зоны репозитория

Репо раскладывается на **пользовательские зоны** (твоё содержимое) и одну **системную зону** `.exocortex/`:

| Зона | Что живёт |
|---|---|
| `concepts/` | личная база концептов — ядро экзокортекса |
| `About-Me/` | модель тебя (стиль, ценности, цели, анти-паттерны); **приватная** |
| `sources/` | входящие материалы (`inbox/`, `mail/`, `calendar/`, сессии) |
| `operation/` | сессии-атомы и дневник (`sessions/`, `daily/`) |
| `projects/` | проектные карточки |
| `areas/` | перспективы-программы (зоны ответственности) |
| `tools/skills/` | навыки — **канон** |
| `.exocortex/` | системная зона: `manifest.yaml`, `memory/`, проекции |

`.exocortex/manifest.yaml` — паспорт репо: агенты, поверхности, карта зон, подписки, `init_state` (прогресс инициации). Агент читает его, чтобы понять устройство твоего экзокортекса.

**Приватность держится на git-правах.** Приватные зоны (`About-Me/`) кладутся в `.gitignore` публичного форка и физически не утекают: у других участников нет доступа к твоему личному репо — только к общему пулу ([M5](M5-bok-and-distill.md)).

### 2.3 `sync-agents`: канон в `tools/skills/`, а `.claude/` — проекция

Критичный принцип, который легко нарушить руками:

- **Канон навыков — в `tools/skills/`.** Единственное место, где навык редактируется.
- **`.claude/`, `.codex/`, `AGENTS.md` — генерируемые проекции.** Их создаёт `/sync-agents` под каждого агента из манифеста. **Руками их не трогать** — правки затрутся при следующем прогоне.

`/sync-agents` берёт канон из `tools/skills/` и раскладывает в agent-specific форматы (`.claude/skills`, `.claude/commands`, `AGENTS.md`, тонкий `CLAUDE.md`). Claude Code подхватывает `.claude/skills` **автоматически**. (Нюансы поверхностей — в [../surfaces.md](../surfaces.md).)

Правило переносимости: изменение, которое должно пережить смену агента, вносится в **канон** (`tools/skills/`), а не в проекцию.

### 2.4 Команды агентного режима

| Команда | Что делает |
|---|---|
| `/init-exocortex` | развернуть/дополнить репо: зоны + `.exocortex/`, проекции агентов |
| `/init-me` | заполнить профиль `About-Me/about-me.md` (интервью или из резюме) |
| `/review-concepts` | извлечь концепт из материала → карточка в `concepts/` |
| `/link-bok` | подключить методологию/пул как git-подписку (submodule) — [M5](M5-bok-and-distill.md) |
| `/update-boks` | подтянуть апдейты подписок (read-only слой; личное не трогается) |
| `/sync-agents` | пересобрать проекции `.claude/`/`AGENTS.md` из канона `tools/skills/` |

`/init-exocortex` — интерактивный поток с грейсфул-прерыванием: прогресс пишется в `.exocortex/manifest.yaml` (`init_state: seed → agents → scaffold → profile → first-concept → complete`). Прервался — при следующем запуске продолжаешь с места, ничего не перезаписывая.

### 2.5 Мост отчуждаемости

Экзокортекс — **самовоспроизводящийся шаблон**. Ты не покупаешь доступ к чужой системе — получаешь seed, который разворачивается в **твой личный git-репо**. Дальше:

- **`git clone` — и знание уехало с тобой.** Репо переносится на второй ноутбук, на сервер, к другому агенту одной командой. Ничего не «остаётся на платформе».
- **`profile_sync` — мост веб ↔ агент.** Профиль (`About-Me/`, «Моя память» веб-режима) синхронизируется между веб-платформой и агентным репо. Начал в вебе ([M3](M3-web-platform.md)) — скачал репо — продолжаешь в агенте, профиль на месте. Режимы не изолированы: это две поверхности одного знания.

Отчуждаемость — не лозунг, а операция: твой когнитивный слой всегда при тебе, потому что он в обычном git.

### 2.6 Лестница зрелости Base → Pro → Team → Master

Экзокортекс растёт по уровням; необязательно начинать с максимума:

| Уровень | Что входит |
|---|---|
| **Base** | концепты (`concepts/`) + навыки (`tools/skills/`) |
| **Pro** | + сессии (`operation/`) + профиль (`About-Me/`) + подписки на методологии (BoK) |
| **Team** | + двусторонний обмен `shared`-концептами в общем пуле |
| **Master** | полный полигон: все зоны, свои расширения, проекции под несколько агентов |

Главный принцип переноса уровня — **«контракт, не реализация».** Гарантируется **контракт зон** (что где живёт по назначению), а не конкретная внутренняя раскладка. Master-инстанс богаче Base, но контракт тот же — поэтому навыки и команды работают одинаково на любом уровне.

## Слой 3 · Техники (применить)

**Развёртывание репо (10 мин):**
- [ ] Установлен git и агент (Claude Code / Cursor / Cowork).
- [ ] Запусти `/init-exocortex`. Ответь на вопросы: агенты (multi-select) → способ (clone публичного seed / offline scaffold) → каталог → имя.
- [ ] Команда развернёт зоны + `.exocortex/manifest.yaml`, сделает первый commit.

**Заполнение профиля (10 мин):**
- [ ] Запусти `/init-me` — заполни `About-Me/about-me.md` в диалоге или из файла-резюме.
- [ ] Проверь, что `About-Me/` в `.gitignore` (если планируешь публичный форк).

**Проекции (2 мин):**
- [ ] Прогони `/sync-agents` — соберутся `.claude/skills|commands`, `AGENTS.md`, тонкий `CLAUDE.md`.
- [ ] Убедись: правил проекции руками не делал; канон — в `tools/skills/`.

**Практика на своём:** заведи первую карточку — положи документ в `sources/inbox/` и запусти `/review-concepts`. Ценность видна в первый час: «из моего материала — моё знание». Пошаговый разбор — в [../01-first-review.md](../01-first-review.md).

## Слой 4 · Кейсы (различить)

- **Кейс А (перенос, ок).** Основатель развернул репо через `/init-exocortex`, работает неделю. Пересел за второй ноутбук: `git clone` своего remote — весь экзокортекс на месте, агент сразу читает `.exocortex/manifest.yaml`.
- **Кейс Б (антипаттерн — правка проекции).** Пользователь поправил навык прямо в `.claude/skills/…`. Следующий `/sync-agents` затёр правку. → Править **канон** в `tools/skills/`, потом `/sync-agents`.
- **Кейс В (мост из веба).** Занятой участник начал в веб-режиме ([M3](M3-web-platform.md)): «Моя память» копилась в графе. Захотел отчуждаемости — скачал репо, `profile_sync` перенёс профиль, дальше работает в Claude Code.
- **Кейс Г (рост по лестнице).** Начал с Base (только `concepts/` + `tools/skills/`). Через месяц добавил `operation/` и профиль — стал Pro. Контракт зон не менялся, команды те же.

## Слой 5 · Границы, связи, эволюция (встроить)

**Антипаттерны:**
- Ручная правка `.claude/`/`AGENTS.md` (проекция затрётся при `/sync-agents`).
- Репо под агрессивным облачным клиентом поверх `.git` (конфликты — см. [M0](M0-workspace.md)).
- Публичный форк без `About-Me/` в `.gitignore` (утечка приватной зоны).
- Ожидание, что смена LLM что-то ломает: движок снаружи и сменный — не ломает.

**Границы режима:**
- Порог входа выше веба: нужен git + агент. Зато — полная отчуждаемость.
- Приватность держится на git-правах, а не на настройках платформы.

**Связи:**
- ← [M0](M0-workspace.md) — где разместить репо и почему латиница в именах.
- ← [M3](M3-web-platform.md) — веб-режим; мост `profile_sync` соединяет его с агентным.
- → [M5](M5-bok-and-distill.md) — как в этот репо приходят BoK (`/link-bok`, `/update-boks`) и как концепт публикуется в пул.
- ↔ [../surfaces.md](../surfaces.md) — из какого агента работать (Claude Code · Cowork · Codex · Obsidian).

**Эволюция:** уровень наращивается по лестнице Base→Pro→Team→Master; контракт зон стабилен, реализация — твоя. LLM снаружи и сменный — репо переживёт смену и агента, и модели.

## Практика (deliverable)

1. **Разверни репо** `personal-cognitive-os` через `/init-exocortex` (clone seed или offline scaffold) — зоны + `.exocortex/manifest.yaml`, первый commit.
2. **Заполни профиль** через `/init-me` → `About-Me/about-me.md`.
3. **Прогони** `/sync-agents` — проверь, что появились проекции `.claude/` и `AGENTS.md`, а канон навыков лежит в `tools/skills/`.
4. (Опционально, мост в [M5](M5-bok-and-distill.md)) заведи первую карточку концепта: `sources/inbox/` → `/review-concepts`.

**Deliverable модуля:** развёрнутый git-репо + заполненный профиль + успешный прогон `/sync-agents` (проекции собраны из канона).

3. Перечисли 4 из зон репо и скажи, какая из них приватная.
4. Что означает «контракт, не реализация» на лестнице Base→Pro→Team→Master?
5. Где находится LLM относительно экзокортекса и почему это делает движок сменным (vendor-independence)?
