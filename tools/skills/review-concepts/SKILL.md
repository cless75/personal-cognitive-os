# Навык: авторский ревью концептов (документ/транскрипт → карточка концепта)

Флагманский навык репозитория. Извлекает из материала кандидатов-концептов и **проводит владельца через авторское ревью**, финализируя только подтверждённое. Принцип: *AI — редактор, не сочинитель; материал = сырьё, не готовый текст; ничего не канонизируется без явного вердикта автора.*

## Вход

- Документ или транскрипт: файл в `sources/inbox/` или другом канале `sources/` (`.md`/`.txt`), путь, или вставленный текст.
- Текст — приоритетный источник (аудио не прогонять заново).

## Протокол

### Шаг 1 — Анализ (📋 покажи, продолжай)
Прочитай материал. Определи источник (что за файл, дата если есть). Найди все места, где владелец формулирует **модель / принцип / различение / метафору / правило**. Выпиши список кандидатов с прямыми цитатами (и таймкодами, если есть).

### Шаг 2 — Классификация (📋)
Сверь каждого кандидата с **уже существующими карточками в `concepts/`** (прочитай их заголовки и определения). Присвой метку:
- `[НОВЫЙ ИНСАЙТ]` — концепта ещё нет.
- `[КОЛЛИЗИЯ]` — расходится с существующей карточкой (предъяви обе формулировки).
- `[УТОЧНЕНИЕ]` — развивает ядро существующей карточки.
- `[OK]` — уже покрыто, ничего не делать.

Покажи таблицу: кандидат · метка · цитата · предлагаемое действие.

### Шаг 3 — Draft-карточка (по одному концепту)
Для каждого `[НОВЫЙ ИНСАЙТ]`/`[УТОЧНЕНИЕ]`/`[КОЛЛИЗИЯ]` собери draft по `concepts/_template-concept.md`:
- `CANONICAL: false`, `status: pending-author-validation`, `version: v1-draft`.
- Ядро в `## Логика модели` помечено `[ТРЕБУЕТ РЕВЬЮ]` + **прямая цитата владельца** (не перефраз!).
- Заполни все секции чернового: Определение · Pattern · Ядро/Взаимосвязи/Границы/Маркеры коллизий · Связи · Эволюция.
- **Сохрани авторский голос:** лексикон, конструкции «A, а не B», конкретику. Не сглаживай острые формулировки, не добавляй generic-связки.

### Шаг 4 — ⛔ Авторское ревью (обязательная остановка)
Покажи draft владельцу и задай **3–7 вопросов** (по слоям, не больше нужного):
1. **Ядро/инварианты:** «Это несжимаемое ядро? Без чего концепт разваливается?»
2. **Границы:** «Где это НЕ работает / антипаттерн?»
3. **Связи/коллизии:** при `[КОЛЛИЗИЯ]` — «слить, развести как cross-ref, или это разные концепты?»
4. **Имя:** «Как назвать? Публичное vs внутреннее имя?»
5. **Присвоение:** «Это твой концепт или заимствованный? (влияет на `Авторство`)»

Затем прогони concept через **4 guard'а качества** и покажи вердикты:
- **category-error** — узел не смешивает измерение и его элемент?
- **forced-symmetry** — не лестница/матрица там, где карта равноправных вариантов?
- **substrate** — не путаем «что» (хранилище/материал) и «как» (принцип)?
- **false-second-axis** — смежный инструмент не превращён во «вторую ось»?

**Жди явных ответов.** По каждому пункту фиксируй: Выполнить / Пропустить / Скорректировать. Без ответа — НЕ финализируй (triangulation ≠ canon без явной валидации автора).

### Шаг 5 — Финализация
После вердикта:
- Сними маркеры `[ТРЕБУЕТ РЕВЬЮ]`.
- `CANONICAL: true`, `status: canon-v1`, `version: v1`, `AVP_date: <сегодня>`.
- Заполни `## Журнал авторского ревью` (по пункту — вердикт; guard'ы — отметки).
- Запиши карточку в `concepts/<kebab-name>.md`. При `[КОЛЛИЗИЯ]` с решением «слить» — обнови существующую карточку (блок «Эволюция»), не плоди дубль.
- Проставь `## Связи с другими концептами` (относительные `[[wikilink]]`).

## Проверка голоса (после каждой правки)
Приём «сказал бы я это так?»: если формулировка звучит как AI-полировка — верни авторскую версию. «Менее гладко» лучше, чем «не моё».

## Выход
Одна или несколько карточек концептов в `concepts/` + короткая сводка «что извлечено, что отклонено, что осталось в очереди».
