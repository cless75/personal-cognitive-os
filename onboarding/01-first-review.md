# Первый авторский ревью — шаг за шагом

Аудитория: тебе комфортно с кодом и git; Claude уже установлен. Цель — за один проход получить первую карточку концепта из своего материала.

## Два входа

### Вход B — быстрый (в Claude Cowork)

Если установлен глобальный навык инициации (см. [surfaces.md](surfaces.md#cowork)):

1. Запусти `/init-exocortex`.
2. Ответь на вопросы (куда развернуть, как тебя зовут) — команда сделает clone/scaffold, персонализирует `About-Me/` и заведёт карту проектов.
3. Переходи к шагу 4.

### Вход A — ручной

1. `git clone <url-репо>` в удобный каталог.
2. Открой папку в **Cowork** или **Obsidian** (см. [surfaces.md](surfaces.md)).
3. Запусти `/init-me` — заполни профиль `About-Me/about-me.md` (в диалоге или из своего файла-резюме).

## Общие шаги

4. **Положи материал.** Скопируй свой документ или транскрипт в `sources/inbox/` (или просто вставь текст в диалог).
5. **Запусти ревью.** `/review-concepts`. Навык:
   - извлечёт кандидатов-концептов и классифицирует их (`[НОВЫЙ ИНСАЙТ]` / `[КОЛЛИЗИЯ]` / `[УТОЧНЕНИЕ]` / `[OK]`);
   - покажет draft-карточку, где ядро — **твоими словами** (цитата из источника, помечено `[ТРЕБУЕТ РЕВЬЮ]`);
   - задаст 3–7 вопросов по слоям (ядро, границы, связи, имя) и прогонит guard'ы качества;
   - **дождётся твоего явного вердикта** по каждому пункту.
6. **Готово.** Карточка появится в `concepts/` со статусом `canon-v1`, `CANONICAL: true` и журналом ревью.

> Это теперь твой репозиторий. Повторяй цикл — `concepts/` растёт в твою личную, проверенную базу концептов.

## Дальше

- `/list-concepts` — обзор того, что накопил.
- `/new-project`, `/start-session`, `/close-session` — веди работу по проектам (карта проектов сама пополняется в `CLAUDE.md`).
