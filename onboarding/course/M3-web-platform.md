# M3 · Веб-платформа: второй режим (обзор)

> **Столп B — работа на платформе.** ~30 мин на чтение. Это **обзорный** модуль: он объясняет второй способ жить в Cognitive OS — веб-режим. **Детальная практика веба — на самой платформе**, а этот курс идёт агентным треком ([M4](M4-agent-mode.md)). Читай M3, чтобы понимать выбор режима и мост между ними. [Карта курса](00-start.md).

---

## Слой 1 · Сущность (узнать)

**Веб-режим Cognitive OS** — работа с твоим знанием через браузер, где граф знаний живёт в **базе данных платформы**. Ты не ставишь агент, не клонируешь репо — открываешь страницу и работаешь.

Три экрана веб-режима:
- **`/solve`** — решение задач: выбираешь Skill, вводишь свою задачу, идёшь по протоколу.
- **`/graph`** — граф: узлы (факты, инсайты, следующие шаги), накопившиеся из твоих сессий.
- **«Моя память»** — твои личные узлы, по умолчанию **включаются в контекст сессии**, чтобы не объяснять себя заново.

Один тезис: **Skill — это не «AI-чат», а протокол с применимым результатом (applied result).** Ты приходишь не поговорить, а получить конкретный следующий шаг и оставить после себя узлы в графе.

---

## Слой 2 · Концепты-принципы (понять)

### 2.1 Skill = протокол с applied result (а не AI-чат)

| | AI-чат | Skill (протокол) |
|---|---|---|
| Что даёт | текст ответа | **применимый результат** (конкретный шаг) |
| Что остаётся | история переписки | **узлы в графе** (fact / insight / next_step) |
| Форма | свободный диалог | структурированный протокол (5 Почему, разбор проблемы, …) |
| Цель | «поговорить» | «продвинуть застрявшую задачу» |

Сессия по Skill не растворяется в чате. Её выводы **материализуются в граф**: факты → узлы `fact`, инсайты → `insight`, следующие шаги → `next_step`. У каждого узла — provenance (откуда пришёл, кем подтверждён). Память накапливается из **проверенного тобой** контента, а не из болтовни.

**Applied result** — момент, когда ты принимаешь предложенный системой конкретный шаг как результат сессии. Создаётся узел `result`. Не «прочитал умный ответ», а «получил шаг, который можно сделать завтра».

### 2.2 DPS-уровни оркестрации (глубина = задача)

| Уровень | Что это | Сколько Skills | Время |
|---|---|---|---|
| **L0 · Retrieval** | достать готовое из памяти/графа | 0 (поиск) | < 5 с |
| **L1 · Quick Solve** | быстро прогнать одну задачу | 1 Skill | минуты |
| **L2 · Structured** | связка нескольких протоколов | 2–3 Skills | дольше |
| **L3 · Deep System** | системный разбор целиком | полный pipeline | сессия |

**Глубина подбирается под задачу, а не наоборот.** Мелкий вопрос — L0/L1; застрявшая многослойная проблема — L2/L3. Зрелость — не «жечь» L3 там, где хватит L1.

### 2.3 Как работает «Моя память»

Петля замкнута в обе стороны:
- **Наполнение (сессия → граф).** Сохранённые выводы пишутся как узлы `fact` / `insight` / `next_step` / `result`. Система разделяет **факты о тебе** (`personal_profile`) и **твои предметные инсайты** (`personal_knowledge`). Правишь вывод — узел обновляется; удаляешь — soft-delete (связи сохраняются).
- **Подача обратно (граф → промпт).** При запуске **любого** Skill «Моя память» по умолчанию **включена**: профиль + релевантные узлы добавляются в контекст, помеченные источником (не смешиваются со знанием BoK).
- **Opt-out.** Переключатель снимаемый — по одной сессии; при этом **запись продолжается** (снятие влияет только на чтение памяти в промпт).

### 2.4 Порог входа: браузер, без установки

Веб-режим — самый низкий порог: нужен только браузер, граф и память на платформе, структура графа **одинакова для всех уровней** (веб не «урезанный»). Чего в веб-режиме **нет** — **отчуждаемости** (скачать свой репо и уйти) и стыковки с локальным агентом. Это следующий шаг по лестнице Base → Pro — и он разбирается в [M4](M4-agent-mode.md).

## Слой 3 · Практика (обзорная)

Детальная практика веб-режима (`/solve`, `/graph`, applied result) выполняется **на платформе**, а не в этом репо. Если у тебя есть доступ к веб-платформе и хочется попробовать второй режим:

1. Открой `/solve`, впиши **свою** застрявшую задачу (не демо), выбери короткий Skill, доведи до **applied result**.
2. Проверь в `/graph`, что появились узлы (`insight` / `next_step` / `result`).

Если ты идёшь чисто агентным треком — этот модуль остаётся **обзорным**: важно понять, что веб и агент — две поверхности одного знания, соединённые мостом (`profile_sync`, [M4](M4-agent-mode.md)). Переходи к [M4](M4-agent-mode.md).

## Слой 4 · Кейсы (различить)

- **Кейс А (правильно, applied result).** Участник вписывает в `/solve` реальную задачу «за месяц ушли 3 сениора», проходит «5 Почему», принимает шаг «провести exit-разбор». Появляется `result` + `next_step`. Задача сдвинулась.
- **Кейс Б (антипаттерн «AI-чат»).** Пользователь относится к `/solve` как к чат-боту: читает ответы, ничего не принимает как результат. Граф пустой. → Skill доводят до applied result.
- **Кейс В (демо вместо своего).** Участник вводит «красивую» учебную задачу. Память наполнилась чужим контекстом, своя задача не сдвинулась. → Всегда бери **свою** задачу.
- **Кейс Г (неверный уровень).** На вопрос «как назвать раздел» человек запускает L3-pipeline и вязнет. → Мелочь — L0/L1; L3 для системных проблем.

## Слой 5 · Границы, связи, эволюция (встроить)

**Антипаттерны:**
- Относиться к `/solve` как к AI-чату (нет applied result → нет памяти).
- Бросать сессию до принятия результата (граф не наполняется).
- Прогонять демо-задачи вместо своих.
- Гнать L3 там, где хватает L1.

**Связи:**
- ← [M1](M1-principles.md) — зачем это (Cognitive OS, 7 компонентов); веб-режим = Session Hub + BoK в БД.
- → [M4](M4-agent-mode.md) — агентный режим, **отчуждаемость** (git-репо), лестница Base→Pro→Team→Master, мост `profile_sync` веб ↔ агент.
- ↕ [M2](M2-four-modes.md) — 4 режима работы с ИИ и Оркестратор как контекст выбора Skill.

**Эволюция:** можно начать в вебе (низкий порог), накопить граф и «Мою память», позже включить агентный режим ([M4](M4-agent-mode.md)). Знание переносимо: накопленное в вебе становится основой личного экзокортекса.
