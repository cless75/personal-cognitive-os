# daily/ — журнал сессий

Здесь навыки `start-session` / `close-session` ведут лёгкий журнал работы по проектам. Один файл на день: `YYYY-MM-DD.md`.

Формат записи (чистый Markdown, без плагинов):

```markdown
## Сессии

### HH:mm — [[projects/<slug>/<slug>]] (старт)
- Цель: …

### HH:mm — [[projects/<slug>/<slug>]] (итог)
- Сделано: …
- Следующий шаг: …
- Статус: продолжу / готово / отложено
```

Журнал — вспомогательный слой. Канон итога по концептам — в `concepts/`, по проекту — в карточке проекта и карте проектов (`CLAUDE.md`).
