# Навык: инициация Personal Cognitive OS (новый репо ИЛИ в существующий проект)

Доводит владельца до первого рабочего цикла. Два режима — выбираются на шаге 1.

## Шаг 1 — Режим (AskUserQuestion)

- **A. Новый репозиторий** — развернуть отдельный чистый Personal Cognitive OS.
- **B. В существующий проект Cowork (in-place)** — добавить экзокортекс в уже открытую папку проекта, ничего не ломая.

Определи контекст: если навык запущен внутри непустой папки с уже существующим содержимым — предложи режим B по умолчанию.

---

## Режим A — Новый репозиторий

1. **Параметры** (AskUserQuestion): способ (clone публичного seed `github.com/cless75/personal-cognitive-os` / scaffold из встроенного шаблона оффлайн) · каталог · имя.
2. **Развёртывание:** `git clone <seed>` (или scaffold структуры). Подставь токены `{{learner}}`/`{{learner_email}}`/`{{date}}`. `git init` + первый commit (если новый).
3. **Персонализация:** запусти логику `init-me` (заполни `About-Me/about-me.md`).
4. **Карта проектов:** убедись, что в `CLAUDE.md` есть пустой блок `projects-map`.
5. **Первый шаг:** предложи положить файл в `sources/` и запустить `/review-concepts`.

---

## Режим B — В существующий проект Cowork (in-place, НЕДЕСТРУКТИВНО)

> Главное правило: **ничего не удаляем и не перезаписываем**. Только добавляем недостающее и дописываем в размеченные блоки.

1. **Скан проекта:** прочитай, что уже есть в папке (файлы, есть ли `CLAUDE.md`, `concepts/`, `.claude/`). Покажи владельцу, что обнаружено.
2. **Подтверждение** (AskUserQuestion): «Добавить структуру экзокортекса в этот проект? Существующие файлы не трогаю.»
3. **Добавь недостающие папки** (только если их нет): `concepts/` (+ `_template-concept.md`, `README.md`), `About-Me/about-me.md`, `sources/` (+ README), `projects/` (+ шаблон, README), `daily/`, `onboarding/`. Существующие одноимённые — НЕ трогай.
4. **CLAUDE.md (слияние, не перезапись):**
   - Если `CLAUDE.md` уже есть — **допиши** в конец секцию «## Personal Cognitive OS» (правила: атомарность концептов, авторское ревью, авторский голос) + блок карты проектов между маркерами `<!-- projects-map:start -->…<!-- projects-map:end -->`. Не трогай существующее содержимое файла.
   - Если нет — создай из шаблона seed.
   - Если маркеры `projects-map` уже присутствуют — не дублируй.
5. **Навыки в существующем проекте:**
   - **Claude Code:** скопируй `.claude/skills/*` и `.claude/commands/*` seed в `.claude/` проекта (только отсутствующие; не перезаписывай чужие навыки).
   - **Cowork:** напомни, что Cowork не читает `.claude/skills` из папки — установи навыки как плагин (`.claude-plugin/`) или ZIP (см. `distributions/README.md`). Если у проекта уже есть свой плагин — добавь навыки в него, не конфликтуя.
6. **Персонализация:** `init-me` — но если в проекте уже есть профиль/контекст, дополни `About-Me/`, не затирая.
7. **Первый шаг:** предложи положить материал в `sources/` и запустить `/review-concepts`.

**Проверка после B:** существующие файлы проекта не изменены (кроме допушенного блока в `CLAUDE.md`); новые папки добавлены; карта проектов инициализирована.

---

## Заметка о среде
- **Claude Code:** репо-локальные навыки в `.claude/skills` подхватываются автоматически.
- **Cowork:** установка навыков — плагин/ZIP (`onboarding/surfaces.md`, `distributions/README.md`).
