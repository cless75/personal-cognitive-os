# M0 · Рабочее пространство: место, конвенция, способ работы

> **Столп B — работа на платформе.** Self-paced: прочитай гайд → сделай практику на своём → пройди self-quiz → зафиксируй deliverable. ~40 мин. Если проходишь курс по порядку — это первый шаг после intake. [Карта курса](00-start.md).

---

## Слой 1 · Сущность (узнать)

**Рабочее пространство (workspace)** — это одно место, где живёт твоё знание и откуда с ним работает ИИ. Не «папка с файлами», а дом твоего Cognitive OS: сюда стекаются сессии, здесь накапливаются концепты, отсюда подключаются методологии (BoK).

Два вопроса, на которые отвечает M0:
1. **Где** этот дом будет жить (и по каким критериям выбрать место).
2. **Как** в нём всё называть, чтобы через полгода не утонуть (конвенция).

Плюс — выбрать **способ работы**: веб-платформа или агент (подробно — в [M3](M3-web-platform.md) / [M4](M4-agent-mode.md)).

## Слой 2 · Концепты-принципы (понять)

### 2.1 Критерии выбора места

Место workspace — твой выбор. Оно **не обязано** быть на каком-то конкретном диске или пути. Критерии:

- **Стабильность.** Локальная папка, которая не переезжает и не пересоздаётся. Любой диск с запасом места.
- **Вне конфликтной облачной синхронизации.** Не класть git-репо внутрь папки, которую агрессивно синхронизирует облако (Dropbox/OneDrive/iCloud поверх `.git` → конфликты и битые состояния). Синхронизация знания идёт через **git**, не через файловый облачный клиент.
- **Git-доступ.** Место, где ты можешь сделать `git clone`/`git commit` (для агентного режима это обязательно).
- **Права и приватность.** Ты владеешь папкой; приватные зоны (`About-Me/`) не утекают в публичные форки.
- **Один workspace-корень.** Если у тебя несколько экзокортексов — держи их как подпапки контейнера, не вкладывай один в другой.

### 2.2 Конвенция именования: почему латиница, а не кириллица

Файлы и папки workspace именуются **латиницей**. Это не вкус, а инженерная необходимость:

- **Кросс-платформенность.** Кириллические имена по-разному кодируются в Windows/macOS/Linux → при переносе репо ломаются.
- **Git.** Юникод-имена git показывает как экранированные последовательности (`\320\...`), diff'ы и `git mv` становятся нечитаемыми.
- **URL-slug.** Имя файла часто становится частью ссылки; кириллица в URL превращается в `%D0%...`.
- **Инструменты и скрипты.** Многие CLI/парсеры спотыкаются о не-ASCII в путях.

**Правило разведения:** **имя файла — латиница**, а **заголовок и H1 внутри — на русском**. Смысл на русском живёт в контенте, стабильность — в имени.

### 2.3 Зоны рабочего пространства (обзор)

Экзокортекс раскладывается по зонам (детально — [M4](M4-agent-mode.md)):
- `concepts/` — личная база концептов (ядро)
- `About-Me/` — модель тебя (приватная)
- `sources/` — входящие материалы (inbox, mail, sessions)
- `operation/` — сессии и дневник
- `projects/` · `areas/` — проекты и перспективы
- `tools/skills/` — навыки (канон)
- `.exocortex/` — системная зона (manifest, memory, projections)

### 2.4 Два способа работы (выбор)

| | Веб-платформа | Агентный режим |
|---|---|---|
| Где живёт | БД платформы | локальный git-репо |
| Инструмент | браузер, `/solve` | Claude Code / Cursor / Cowork |
| Порог входа | низкий | нужен git + агент |
| Отчуждаемость | нет | да (`git clone` и ушёл) |

Не обязательно выбирать «навсегда» — можно начать с веба ([M3](M3-web-platform.md)) и позже перейти в агентный режим ([M4](M4-agent-mode.md)). Мост между ними существует. Этот курс идёт **агентным** треком — практика ниже под него.

## Слой 3 · Техники (применить)

**Чеклист выбора места (5 мин):**
- [ ] Выбрал локальную папку с запасом места, вне облачной синхронизации поверх `.git`.
- [ ] Проверил, что можешь сделать там `git clone`/`git commit`.
- [ ] Решил, что идёшь агентным треком (или совмещаешь с вебом).

**Чеклист именования (3 мин):**
- [ ] Имена файлов/папок — латиница.
- [ ] Заголовок и H1 внутри — на русском.
- [ ] Понял разведение «имя латиницей / смысл по-русски».

**Практика на своём:** выбери и зафиксируй место, куда развернёшь свой репо `personal-cognitive-os` (это сделаешь в [M4](M4-agent-mode.md) через `/init-exocortex`). Пробегись по [../surfaces.md](../surfaces.md) — реши, из какого агента будешь работать.

## Слой 4 · Кейсы (различить)

- **Кейс А (агентный, ок).** Основатель кладёт репо в `~/work/my-cogos`, синхронизирует через GitHub, приватный `About-Me/` в `.gitignore`. Переносит на второй ноутбук `git clone` — всё на месте.
- **Кейс Б (антипаттерн).** Пользователь создаёт репо внутри синхронизируемой облаком папки. Облако конфликтует с `.git`, история ломается. → Вынести из-под облачного клиента, синхронизировать git'ом.
- **Кейс В (веб-старт).** Занятой участник не хочет ставить агент — начинает в вебе (`/solve`), «Моя память» копится в графе. Позже, когда захочет отчуждаемости, скачивает свой репо (мост в [M4](M4-agent-mode.md)).

## Слой 5 · Границы, связи, эволюция (встроить)

**Антипаттерны:**
- Репо под агрессивным облачным клиентом (конфликты git).
- Кириллица в именах файлов/путей (ломает перенос и инструменты).
- Несколько экзокортексов, вложенных друг в друга (нет единого корня).

**Связи:**
- → [M1](M1-principles.md) — зачем этот дом нужен (Cognitive OS как расширение мышления).
- → [M3](M3-web-platform.md) (веб) / [M4](M4-agent-mode.md) (агент) — как работать выбранным способом.
- → [M5](M5-bok-and-distill.md) — как в этом доме появляются концепты и BoK.

**Эволюция:** место и способ можно менять; знание переносимо (git). Начать просто — усложнять по лестнице Base→Pro ([M4](M4-agent-mode.md)).

## Практика (deliverable)

1. Зафиксируй **место** будущего репо (путь) и **способ** (агент / агент + веб).
2. Усвой конвенцию: имя файла — латиница, заголовок — русский.
3. Пройди **Cognitive OS baseline** (диагностика 4 квадрантов) — это твоя точка отсчёта; в [M5](M5-bok-and-distill.md) сравнишь delta. Полную диагностику проходишь в [M1](M1-principles.md).

**Deliverable модуля:** выбранное место + понятая конвенция + сохранённый baseline (4 квадранта + dominant gap).

## Self-quiz (free-text, без авто-оценки; цель ≥3/5)

1. Назови 2 критерия, по которым ты выбрал место workspace.
2. Почему имя файла — латиница, а заголовок — русский? Дай одну причину для каждого.
3. Как звучит правило разведения «имя / смысл» в конвенции именования?
4. Чем веб-режим отличается от агентного одним словом про отчуждаемость?
5. Какой у тебя dominant gap по 4 квадрантам Cognitive OS?
