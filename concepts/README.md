# concepts/ — твой репозиторий концептов

Здесь живут **карточки концептов** — атомарные единицы твоего продуманного знания. Один файл = один концепт = одно ядро смысла. Это не заметки и не свалка идей: каждая карточка прошла **авторское ревью** и подтверждена тобой как твоя.

## Как появляется карточка

Не пишешь вручную. Кладёшь материал (документ/транскрипт) в `sources/inbox/`, запускаешь `/review-concepts` — навык извлекает кандидатов, показывает draft, задаёт 3–7 вопросов и проводит через авторское ревью. После твоего явного вердикта draft становится каноном.

## Статусы (frontmatter)

| Поле | Draft | Канон |
|---|---|---|
| `CANONICAL` | `false` | `true` |
| `status` | `pending-author-validation` | `canon-v1` |
| `AVP_date` | пусто | дата ревью |

В теле draft ядро помечено `[ТРЕБУЕТ РЕВЬЮ]` с прямой цитатой из источника — чтобы ты видел свои слова до ответа на вопросы (иначе легко «подписаться» под чужой формулировкой).

## Структура карточки (5 слоёв)

Секции = проекция 5 слоёв Learning Object (атомарная единица знания):

1. **Определение** (одна строка) — слой «Сущность».
2. **Какую задачу решает (Pattern)** — контекст применения.
3. **Логика модели** — Ядро (инварианты) · Ключевые взаимосвязи · Границы применимости · Маркеры коллизий.
4. **Связи с другими концептами** — граф.
5. **Эволюция** + **Журнал авторского ревью** — история и вердикты.

Шаблон — [`_template-concept.md`](_template-concept.md).

## Guard'ы качества (при ревью)

Навык прогоняет каждый концепт через 4 проверки:
- **category-error** — не смешаны ли в одном узле измерение и его элемент.
- **forced-symmetry** — не рисуешь ли лестницу/матрицу там, где карта равноправных вариантов.
- **substrate** — не путаешь ли «что» (хранилище/материал) и «как» (принцип).
- **false-second-axis** — не превращаешь ли смежный инструмент во «вторую ось».
