# Модель экзокортекса — зоны, принципы, лестница

Два компонента: **LLM** (внешний сменный движок резонинга) ↔ **Экзокортекс** (этот репозиторий — твой когнитивный слой). Всё ниже — внутреннее устройство второго.

## 7 зон + системная

| Зона | Функция | Что внутри |
|------|---------|------------|
| `About-Me/` | Self-model: кто владелец | профиль, ценности, (Pro+: baseline-диагностика) |
| `concepts/` | Знание — ядро ценности | карточки концептов (одна = одно ядро смысла) |
| `operation/` | Операционка во времени | `sessions/` — сессии-атомы (провенанс) · `daily/` — журнал дней. Сессия ≠ журнал: связь ссылкой |
| `sources/` | Вход (мультиканальный) | `inbox/` документы · `mail/` · `calendar/` · `team-sessions/` встречи · `ai-sessions/` логи ИИ-диалогов. Base = дропзоны (кладёшь руками); коннекторы — Pro+ |
| `projects/` | Активные дела | одна папка = один проект (PARA-логика внутри) |
| `areas/` | Направления жизни/деятельности | долгоживущие области без даты окончания |
| `tools/` | Мастерская (agent-agnostic) | `skills/` — **канон навыков** · `mcp/` — конфиги · `bok-adaptation/` — внешнее знание на адаптации |
| `.exocortex/` | Системная зона (seed-managed) | manifest · эта модель · реестр агентов · шаблоны проекций |

## Принципы

- **Один экзокортекс — много агентов.** Канон навыков в `tools/skills/`; `.claude/`, `.claude-plugin/` (Cowork), `.codex/` (skills + config.toml, DM-EXO-10), `AGENTS.md`, `.cursor/` — генерируемые проекции (навык `sync-agents`). Смена агента не меняет данные и канон.
- **Self-model ≠ Areas.** `About-Me/` — модель субъекта, не «одна из областей»; живёт в корне.
- **Сессия ≠ журнал дня.** Сессия — атом с провенансом (что откуда взялось); журнал — хронология. Обе внутри `operation/`.
- **Ничего не канонизируется без вердикта владельца.** Draft → канон только через авторское ревью.
- **Автономность.** Всё работает офлайн: чистый Markdown + git. Подписка Платформы (Pro+) даёт обновления и сервисы, не работоспособность.
- **Ядро / расширение.** Переносится контракт (форма данных + протокол), не реализация. Богатые надстройки (Obsidian, ASR, statusline) — опциональны.

## Эволюционная лестница (DM-EXO-8)

`Base` (концепты + навыки; режим Исполнение) → `Pro` (+ сессии, профиль-диагностика, подписки `boks/`; все 4 режима) → `Team` (+ общий пул, `visibility`, роли) → `Master` (+ портфель, упаковка своих S-BoK). Уровень = поле `tier` в манифесте; один репозиторий переживает апгрейды.
