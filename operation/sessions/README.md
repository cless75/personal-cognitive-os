# operation/sessions/ — сессии-атомы

Каждая значимая рабочая сессия — отдельный файл-атом с провенансом: видно, **что откуда взялось** (какие концепты и решения родились из какой работы).

Имя файла: `S{YYYYMMDD}T{HHMM}-{slug}.md` (например `S20260713T0930-first-concepts.md`).

Шаблон:

```markdown
---
SessionId: S20260713T0930-first-concepts
date: 2026-07-13
project: <slug или ->
source: <канал sources/ или диалог>
status: open | closed
---

# Сессия: <о чём>

## Цель

## Результат
- концепты: [[../../concepts/<карточка>]]
- решения / следующий шаг:
```

## Правила

- Сессию заводят/закрывают навыки `/start-session` и `/close-session`; можно и руками — формат выше.
- **Сессия ≠ журнал дня.** Хронология дня — в `../daily/`; сессия ссылается на день, день — на сессию.
- Провенанс: карточка концепта, рождённая в сессии, получает `source_session:` с её SessionId.
