# Навык: инициация Personal Cognitive OS (новый репо ИЛИ в существующий проект)

Доводит владельца до первого рабочего цикла. Два режима — выбираются на шаге 1.

## Шаг 1 — Режим (AskUserQuestion)

- **A. Новый репозиторий** — развернуть отдельный чистый Personal Cognitive OS.
- **B. В существующий проект Cowork (in-place)** — добавить экзокортекс в уже открытую папку проекта, ничего не ломая.

Определи контекст: если навык запущен внутри непустой папки с уже существующим содержимым — предложи режим B по умолчанию.

---

## Режим A — Новый репозиторий

1. **Параметры** (AskUserQuestion): способ (clone публичного seed `github.com/cless75/personal-cognitive-os` / scaffold из встроенного шаблона оффлайн) · каталог · имя.
2. **Развёртывание:** `git clone <seed>` (или scaffold структуры). Подставь токены `{{learner}}`/`{{learner_email}}`/`{{date}}`. `git init` + первый commit (если новый).
3. **Персонализация:** запусти логику `init-me` (заполни `About-Me/about-me.md`).
4. **Карта проектов:** убедись, что в `CLAUDE.md` есть пустой блок `projects-map`.
5. **Первый шаг:** предложи положить файл в `sources/` и запустить `/review-concepts`.

---

## Режим B — Внедрить seed в существующий проект (in-place, НЕДЕСТРУКТИВНО)

> Одной командой: сам подтяни seed и встрой только **seed-managed** части. Главное правило: **ничего не удаляем и не перезаписываем**; только добавляем недостающее и дописываем в размеченные блоки.

1. **Скан проекта:** прочитай, что уже есть в текущей папке (есть ли `CLAUDE.md`, `concepts/`, `.claude/`, свой плагин). Покажи владельцу, что обнаружено, и куда встраиваем (текущий каталог).
2. **Подтверждение** (AskUserQuestion): «Внедрить экзокортекс в этот проект? Существующие файлы не трогаю, добавлю недостающее.»
3. **Подтяни seed во временный каталог:**
   `git clone --depth 1 https://github.com/cless75/personal-cognitive-os <tmp>`
   (если git/сети нет — используй встроенный шаблон навыка как источник).
4. **Встрой seed-managed части** из `<tmp>` в проект (копируй ТОЛЬКО отсутствующее, не перезаписывай существующее одноимённое):
   - Навыки/команды/плагин: `.claude/skills/*`, `.claude/commands/*`, `.claude-plugin/`.
   - Шаблоны и доки: `concepts/_template-concept.md`, `concepts/README.md`, `projects/_template-project.md`, `projects/README.md`, `sources/README.md`, `daily/README.md`, `onboarding/*`, `distributions/*`, `DEPLOY.md`, `START-HERE.md`.
   - **НЕ копируй** пользовательские контейнеры seed как есть — только создай пустые, если их нет: `concepts/` (кроме шаблона), `About-Me/about-me.md` (из шаблона), `sources/`, `projects/`, `daily/`.
5. **CLAUDE.md (слияние, не перезапись):**
   - Если `CLAUDE.md` уже есть — **допиши** в конец секцию «## Personal Cognitive OS» (правила: атомарность концептов, авторское ревью, авторский голос) + блок карты проектов между маркерами `<!-- projects-map:start -->…<!-- projects-map:end -->`. Существующее содержимое не трогай. Если маркеры уже есть — не дублируй.
   - Если нет — создай из seed.
6. **Убери временный каталог** (`<tmp>`).
7. **Среда навыков:**
   - **Claude Code:** скопированные `.claude/skills` подхватываются автоматически.
   - **Cowork:** напомни — установи навыки плагином (`.claude-plugin/`) или ZIP (`distributions/README.md`); если у проекта уже свой плагин, добавь навыки в него без конфликта.
8. **Персонализация:** запусти `init-me`; если в проекте уже есть профиль/контекст — дополни `About-Me/`, не затирая.
9. **Первый шаг:** предложи положить материал в `sources/` и запустить `/review-concepts`.

**Проверка после B:** существующие файлы проекта не изменены (кроме дописанного блока в `CLAUDE.md`); недостающие папки и навыки добавлены; карта проектов инициализирована; `<tmp>` удалён.

---

## Заметка о среде
- **Claude Code:** репо-локальные навыки в `.claude/skills` подхватываются автоматически.
- **Cowork:** установка навыков — плагин/ZIP (`onboarding/surfaces.md`, `distributions/README.md`).
